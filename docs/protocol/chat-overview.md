# Chat Protocol Overview

This protocol is the shared contract for the desktop Tauri client, Flutter mobile client, and FastAPI backend. It keeps chat, group chat, talk rooms, wallpaper tasks, and device commands aligned across clients.

## Scope

Included in the current phase:

- Direct conversations
- Group conversations
- Conversation list and unread counts
- Text messages
- Message acknowledgements, read state, and revoke state
- Talk room lifecycle for voice intercom
- Wallpaper management task lifecycle
- Remote shutdown command lifecycle

Explicitly out of scope:

- Remote desktop control
- Screen sharing
- Mouse or keyboard control
- Recording audit

## Identity

All API resources use server-generated stable ids:

```text
userId: u_...
conversationId: conv_...
messageId: msg_...
groupId: group_...
talkRoomId: talk_...
fileObjectId: file_...
wallpaperTaskId: wallpaper_...
deviceCommandId: device_cmd_...
```

Clients may generate temporary ids for optimistic UI:

```text
clientMsgId: client UUID generated by the sender
clientRequestId: client UUID generated for idempotent commands
```

## Ordering

Messages must include both:

- `serverSeq`: global monotonically increasing sequence for sync.
- `conversationSeq`: monotonically increasing sequence inside one conversation.

Clients sync by cursor instead of pulling the full history every time.

## Time

All server timestamps are ISO 8601 UTC strings. Clients may display localized labels.

## Transport

- HTTP is used for initial list/detail loading and command requests.
- WebSocket is used for realtime message, unread, presence, talk room, wallpaper, and device command events.
- RTC audio tokens are generated only by the backend.
- OSS upload credentials or upload URLs are generated only by the backend.

## Desktop Window Rule

Only the course schedule widget can attach to the desktop wallpaper/icon layer. Chat and all management windows are normal application windows.
