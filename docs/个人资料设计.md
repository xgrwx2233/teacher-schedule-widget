下面是结合 `docs\个人资料设计.md` 最新批注后的新版 plan。范围只包含 UI/UX 与功能实现规划，不写代码。

**总体结论**
下一阶段先做“个人资料 + 好友资料 + 搜索用户/群”的基础能力，然后再做好友申请、好友通知、联系人列表。临时的“输入云端用户 ID 发起单聊”直接删除，不再保留到正式 UI。

资料字段收敛为：

```text
头像
昵称
简介 / 个签
在线状态
```

不显示：

```text
学校
科组
手机号后四位
云端 userId
```

单聊顶部保留：

```text
对讲
...
```

删除单聊顶部的：

```text
远程壁纸管理
远程关闭电脑
远程控制电脑
```

远程壁纸管理、远程关闭电脑等后续迁移到群聊能力中。

**阶段 1：个人资料基础能力**
后端新增个人资料模型，建议独立 `user_profiles` 表，避免继续把所有资料字段塞进 `users`：

```text
user_id
nickname
avatar_url / avatar_object_id
bio
created_at
updated_at
```

接口：

```text
GET /profile/me
PUT /profile/me
GET /profile/users/{userId}
GET /profile/search?keyword=...&scope=all|users|groups
```

规则：
- 注册后自动初始化资料。
- 默认昵称可以是“用户 + 短随机号”，但 UI 不显示手机号后四位。
- 昵称允许重复。
- 自己只能编辑头像、昵称、简介。
- 非好友可查看基础资料：头像、昵称、简介、在线状态。
- 后续头像上传接 OSS，现在可先支持默认头像或本地/远程 URL 占位。

桌面 UI：
- 点击左侧自己的头像，打开独立普通窗口“个人资料编辑”。
- 参考：`UI参考图\桌面端\14、桌面端个人资料编辑窗口.png`
- 该窗口是普通窗口，不是聊天主面板右侧区域。
- 内容包括头像编辑、昵称输入、简介输入、保存、取消。
- 不出现学校、科组、手机号、userId。

**阶段 2：好友资料展示**
好友资料分两种打开方式：

1. 从联系人列表点击好友卡片：
   - 右侧主区域切换为好友资料页。
   - 中间联系人列表中该好友卡片保持选中状态。

2. 从聊天气泡头像、聊天顶部好友头像点击：
   - 弹出独立普通窗口显示好友资料。
   - 不改变当前聊天主窗口状态。

好友资料 UI 参考：

```text
UI参考图\桌面端\2、好友个人资料.png
```

展示内容：

```text
头像
昵称
在线状态
简介 / 个签
发消息按钮
```

不显示编辑入口。好友资料页不是“我的资料编辑页”。

点击“发消息”：
- 调用或复用 `POST /chat/conversations/direct`
- 切回消息页
- 打开对应单聊

**阶段 3：搜索窗口**
入口：
- 聊天窗口中间消息列右上角 `+`
- 弹出菜单参考：`qq消息列添加好友菜单栏.png`

菜单项：

```text
添加好友
创建群聊
添加群
```
（批注，添加好友和添加群放在一起，叫做“添加好友/群”，因为添加好友/群弹出的搜素窗口是相同的。）
点击“添加好友”或“添加群”：
- 打开综合搜索普通窗口。
- 参考：
  - `qq添加好友综合搜索界面全部.png`
  - `qq添加好友综合搜索界面用户.png`
  - `添加好友高级搜索页.png`

搜索窗口筛选标签：

```text
全部
用户
群聊
```

删除推荐、游戏等无关标签。

搜索规则：
- 只有一个搜索输入框。
- 程序判断 keyword 是否为合法手机号。
- 如果是合法手机号：
  - 先做手机号精确搜索。
  - 再做昵称模糊搜索。
  - 合并结果去重。
- 如果不是合法手机号：
  - 只做昵称模糊搜索。
- 群聊搜索后续按群名模糊搜索。

用户搜索结果卡片：
```text
头像
昵称
简介 / 个签
在线状态
右侧操作按钮
```

操作按钮根据关系状态变化：

```text
添加
等待验证
已添加
重新申请
```

状态设计：
- 非好友、无申请：显示“添加”
- 已发送申请、对方未处理：显示“等待验证”或“等待通过”，按钮禁用
- 已是好友：显示“已添加”，按钮禁用；可点击卡片查看好友资料
- 曾被拒绝：显示“重新申请”
- 对方是自己：显示“自己”，按钮禁用
（批注，搜素出来的所有好友都可以查看好友资料，但是只有已是好友的好友资料中才有“发消息按钮”，非好友不能发消息。）
不显示手机号后四位、学校、科组、userId。

**阶段 4：添加好友申请**
点击搜索结果中的“添加”：
- 打开添加好友验证普通窗口。
- 参考：`UI参考图\桌面端\4、添加好友.png`
- 删除左侧搜索列和搜索结果，只保留右侧验证卡片。

验证窗口内容：

```text

对方昵称
验证消息输入框

取消
```

发送成功后的过渡：
- 关闭添加好友验证窗口。
- 回到综合搜索窗口。
- 对应用户卡片状态改为“等待验证”或“等待通过”。
- 搜索窗口中间显示一个轻提示 tip，例如“好友申请已发送”。
- 不立即进入聊天。

后端接口：

```text

GET /friends/requests
POST /friends/requests/{id}/accept
POST /friends/requests/{id}/reject
GET /friends
DELETE /friends/{friendUserId}
```

好友申请字段：

```text

from_user_id
to_user_id
message
status: pending / accepted / rejected
created_at
handled_at
```

**验收标准**
个人资料：
- 点击自己的头像打开独立个人资料编辑窗口。
- 可编辑头像、昵称、简介。
- 保存后聊天头像、会话、好友资料同步更新。

好友资料：
- 从联系人列表点击好友，右侧显示好友资料。
- 从聊天头像点击好友，弹独立好友资料窗口。
- 好友资料无编辑入口。
- 好友资料有“发消息”。

搜索：
- 一个输入框同时支持手机号和昵称。
- 合法手机号先精确搜索，再昵称模糊搜索。
- 非手机号只昵称模糊搜索。
- 搜索结果不显示手机号后四位、学校、科组、userId。
- 搜索结果根据关系状态显示正确按钮。

好友申请：
- 发送申请后验证窗口关闭。
- 回到搜索窗口。
- 对应卡片变为“等待验证/等待通过”。
- 对方好友通知可见。
- 同意后双方联系人列表出现对方。

聊天合并：
- `+` 不再出现 userId 输入。
- 单聊顶部保留“对讲”和“...”。
- 点击好友资料里的“发消息”能进入单聊。
- 现有文本消息、WebSocket、未读、置顶、删除继续正常。

**验收标准**
个人资料：
- 点击自己的头像打开独立编辑窗口。
- 可编辑头像、昵称、简介。
- 保存后聊天头像、会话、好友资料同步更新。

好友资料：
- 联系人列表点击好友，右侧显示资料。
- 聊天头像点击好友，弹独立资料窗口。
- 非好友资料页没有“发消息”。
- 已是好友资料页有“发消息”。

搜索：
- 一个输入框支持手机号和昵称。
- 合法手机号先精确搜，再昵称模糊搜。
- 非手机号只昵称模糊搜。
- 搜索结果不显示手机号后四位、学校、科组、userId。
- 搜索结果可查看资料，并按关系状态显示按钮。

好友申请：
- 发送申请后验证窗口关闭。
- 回到搜索窗口。
- 对应卡片变为等待状态。
- 对方好友通知可见。
- 同意后双方联系人列表出现对方。

聊天合并：
- `+` 不再出现 userId 输入。
- 单聊顶部保留“对讲”和“...”。
- 好友资料“发消息”可进入单聊。
- 现有文本消息、WebSocket、未读、置顶、删除继续正常。
