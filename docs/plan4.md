# 教师助手：视频、大文件上传管理与网盘功能最终设计方案

## 0. 产品定位

本功能不是单纯的“发送文件”，而是要形成一个完整的文件流转闭环：

```text
聊天发送文件 / 视频
        ↓
上传 / 下载传输管理
        ↓
聊天消息卡片展示
        ↓
预览 / 下载 / 转发 / 存入网盘
        ↓
我的网盘 / 群网盘统一沉淀
```

产品目标：

```text

传输负责大文件过程管理
网盘负责长期资料沉淀
```

整体设计要接近 QQ 的成熟 IM 体验：入口清晰、操作克制、右键能力完整、界面不臃肿。

---

# 一、整体信息架构

## 1. 左侧主导航

左侧主导航保留：

```text
消息
好友
网盘
设置
```

其中：

* 「消息」负责好友聊天、群聊、会话列表。
* 「好友」负责好友关系、联系人、好友申请。
* 「网盘」只进入「我的网盘」。
* 「设置」负责全局设置。

不要在左侧「网盘」里放：

```text
最近文件
群网盘
传输列表
回收站
```

原因：

1. 好友之间的上传下载，应该在好友聊天窗口里处理。
2. 群文件属于群上下文，应该在群聊天窗口里处理。
3. 传输管理属于当前会话操作，不应该混进我的网盘。
4. 左侧「网盘」只承载个人长期文件资产，更干净。

# 二、三类文件空间

最终系统里存在三类文件空间：

## 1. 聊天文件

来源：

```text
好友聊天发送文件
群聊发送文件
聊天拖拽上传
聊天工具栏上传
```

特点：

* 默认只存在于聊天消息中。
* 不自动进入我的网盘。
* 不自动进入群网盘。
* 可通过右键菜单手动存入网盘。

## 2. 我的网盘

入口：

```text
左侧主导航 → 网盘
```

用途：

* 用户个人长期文件管理。
* 可以新建文件夹。
* 可以上传文件和文件夹。
* 可以排序、搜索、切换视图。
* 可以把网盘文件转发给好友或群聊。

## 3. 群网盘

入口：

```text
群聊窗口顶部标题栏 → 群网盘
```

位置：

```text
群名称                         [对讲] [群网盘] [...]
```

用途：

* 群内长期教学资料沉淀。
* 适合课件、教学视频、群公告附件、班级资料。
* 权限由群主 / 管理员控制。
* 不在左侧「网盘」里单独展示。

# 三、聊天上传入口设计

## 1. 工具栏上传

聊天输入区工具栏保留文件入口。

推荐工具栏结构：

```text
表情  文件  图片/视频  截图  语音  历史
```

其中：

| 入口    | 作用                |
| ----- | ----------------- |
| 文件    | 选择任意文件            |
| 图片/视频 | 选择图片和视频，自动按媒体消息展示 |
| 拖拽上传  | 直接将桌面文件拖入当前聊天     |

如果用户通过「文件」选择 `.mp4`、`.mov` 等视频格式，系统仍然自动识别为视频消息，而不是普通文件消息。

---

## 2. 拖拽上传

用户将文件拖入聊天消息区域时，出现轻量 Dropzone 遮罩。

视觉：

```text
┌──────────────────────────────────────┐
│                                      │
│              释放文件发送             │
│        支持图片、视频、文档、大文件     │
│                                      │
└──────────────────────────────────────┘
```

样式建议：

```css
.chat-dropzone {
  background: rgba(47, 123, 255, 0.08);
  border: 2px dashed rgba(47, 123, 255, 0.55);
  border-radius: 16px;
  backdrop-filter: blur(2px);
}
```

交互规则：

1. 拖入聊天区，显示遮罩。
2. 拖出聊天区，遮罩消失。
3. 松开鼠标，文件进入发送队列。
4. 多文件拖拽时，按文件列表依次创建上传任务。
5. 如果拖入文件夹，提示是否作为文件夹上传。

# 四、发送前确认规则

## 1. 小文件直接发送

小文件不需要弹窗确认。

流程：

```text
选择文件 / 拖入文件
        ↓
聊天区立即生成文件消息卡片
        ↓
显示上传进度
        ↓
上传完成
```

推荐阈值：

```text
小文件：< 100MB
大文件：≥ 100MB
超大文件：≥ 1GB
```

## 2. 大文件轻确认

当文件大于等于 100MB 时，弹出轻量确认弹窗。

```text
发送大文件

实验演示视频.mp4
1.2 GB

发送到：美女老师群

[取消]    [发送]
```

如果是多个大文件：

```text
发送 3 个大文件

总大小：2.4 GB
发送到：美女老师群

[取消]    [发送]
```

不要默认勾选「保存到网盘」。

可选项：

```text
□ 同时存入我的网盘
```

群聊中，群主 / 管理员可看到：

```text
□ 同时存入群网盘
```

默认不勾选，避免网盘污染。

# 五、视频消息设计

视频消息不要做成普通文件卡片，应使用独立视频样式。

## 1. 上传完成状态

```text
┌──────────────────────────────┐
│                              │
│          视频封面图            │
│             ▶                │
│                              │
│  03:25              128 MB   │
└──────────────────────────────┘
```

视觉规范：

```text
宽度：240px - 320px
比例：16:9
圆角：12px
背景：视频封面 / 黑色占位
中心：播放按钮
左下角：视频时长
右下角：文件大小
```

## 2. 上传中状态

```text
┌──────────────────────────────┐
│          视频封面图            │
│                              │
│  上传中 54%                  │
│  ━━━━━━━━━━━━━━━━░░░░         │
│  3.2 MB/s · 剩余 1分钟        │
│  [暂停] [取消]                │
└──────────────────────────────┘
```

## 3. 视频右键菜单

视频卡片默认保持干净，不直接堆按钮。操作放在右键菜单或 hover 更多菜单。

右键菜单：

```text
播放
下载
另存为
转发
存入我的网盘
存入群网盘
复制文件名
删除
```

规则：

* 私聊中显示「存入我的网盘」。
* 群聊中，如果用户有权限，显示「存入群网盘」。
* 如果用户无群网盘写入权限，则隐藏「存入群网盘」。
* 已经在网盘里的文件，显示「在网盘中查看」。

| 功能         | 要求                         |

# 九、大文件传输管理

大文件上传下载不能只靠聊天气泡里的进度条。必须有「当前会话传输管理」。

## 1. 入口位置

在好友聊天窗口和群聊窗口顶部工具区增加：

```text
[对讲] [传输] [...]
```

或者图标形式：

```text
麦克风 对讲    ⇅ 传输    更多
```

推荐按钮名称：

```text
传输
```

不要叫：

```text
文件中心
上传管理
下载管理
```

「传输」最简洁，也最符合 QQ / IM 心智。

## 2. 传输窗口

点击「传输」后，打开右侧抽屉或居中弹窗。推荐右侧抽屉，因为不会打断聊天。

```text
传输

[上传中] [下载中] [已完成]

实验演示视频.mp4
1.2 GB · 上传中 64% · 4.5 MB/s · 剩余 2分钟
━━━━━━━━━━━━━━━━░░░░
[暂停] [取消]

课件合集.zip
上传失败，网络中断
[重试] [取消]
```

## 3. 传输状态

```text
等待中
上传中
下载中
暂停
失败
已完成
已取消
秒传完成
```

## 4. 必须支持的能力

```text
分片上传
断点续传
暂停 / 继续
失败重试
取消上传
上传速度
剩余时间
秒传
同文件去重
```

## 5. 传输入口红点 / 进度提示

当有传输任务时，顶部「传输」按钮可以显示小状态：

```text
传输 2
```

或：

```text
传输 · 64%
```

有失败任务时显示红点。

# 十、下载体验

## 1. 小文件下载

小文件点击卡片即可下载到系统 Downloads。

推荐规则：

```text
小文件：< 100MB
点击文件卡片 → 自动下载到 Downloads
```

下载完成后 toast：

```text
已下载到 Downloads
```

toast 可提供：

```text
打开文件
打开文件夹
```

## 2. 大文件下载

大文件不要直接默认下载，右键菜单提供：

```text
下载
另存为
存入网盘
```

点击「下载」：

```text
默认保存到 Downloads
```

点击「另存为」：

```text
弹出系统文件选择器，让用户选择保存位置
```

## 3. 下载设置

设置中增加：

```text
下载设置

默认保存位置：Downloads
下载前询问保存位置：开启 / 关闭
下载完成后通知：开启 / 关闭
```

默认：

```text
默认保存位置：Downloads
下载前询问保存位置：关闭
```

# 十二、群网盘设计

## 1. 群网盘入口

不放在左侧「网盘」里。

放在群聊天窗口顶部标题栏右侧：

```text
美女老师群（5）                         [对讲] [群网盘] [...]
```

也可以用图标 + 文字：

```text
[📁 群网盘]
```

如果空间不足，显示图标，hover tooltip：

```text
群网盘
```

## 2. 群网盘页面打开方式

点击「群网盘」后，推荐打开右侧大抽屉或独立内容面板。

推荐：

```text
右侧抽屉 / 当前聊天区覆盖面板
```

不要跳出主窗口。

## 3. 群网盘页面结构

```text
┌──────────────────────────────────────────────────┐
│ 美女老师群 · 群网盘                               │
│ 搜索群文件...      新建文件夹  上传文件  上传文件夹 │
├──────────────────────────────────────────────────┤
│ 全部  视频  文档  图片  压缩包  其他              │
├──────────────────────────────────────────────────┤
│ 名称                 上传者       时间      大小  │
│ 📁 课件资料           大美女1号    6月23日   --    │
│ 🎬 实验演示.mp4       大美女1号    6月21日   1.2GB │
│ 📄 群通知.docx        大美女2号    6月20日   2.1MB │
└──────────────────────────────────────────────────┘
```

## 4. 群网盘不自动沉淀聊天文件

聊天中发送的文件不会自动进入群网盘。

只有两种情况会进入群网盘：

```text
1. 群主 / 管理员主动上传到群网盘
2. 群主 / 管理员右键聊天文件 → 存入群网盘
```

这样群网盘才会干净。

# 十三、群网盘权限

## 1. 默认权限

```text
群主 / 管理员：上传、删除、重命名、移动、转发、下载

```

## 2. 权限表

| 角色    | 上传   | 新建文件夹 | 删除  | 重命名 | 移动  | 下载  | 转发  |
| ----- | ---- | ----- | --- | --- | --- | --- | --- |
| 群主    | 可以   | 可以    | 可以  | 可以  | 可以  | 可以  | 可以  |
| 管理员   | 可以   | 可以    | 可以  | 可以  | 可以  | 可以  | 可以  |
| 普通成员  | 看群设置 | 看群设置  | 不可以 | 不可以 | 不可以 | 可以  | 可以  |
| 未入群用户 | 不可以  | 不可以   | 不可以 | 不可以 | 不可以 | 不可以 | 不可以 |

## 3. 群设置中增加

```text
群网盘权限

谁可以上传：
- 仅群主和管理员
- 所有群成员

谁可以删除：
- 仅群主
```

```text

成员可以下载 / 转发
```

# 十四、从网盘转发文件

这是网盘和聊天联动的核心。

## 1. 我的网盘转发

我的网盘文件右键：

```text
转发
```

打开转发选择器：

```text
转发文件

实验演示视频.mp4
1.2 GB

搜索好友或群聊...

最近聊天
好友
群聊

[取消] [发送]
```

发送后，聊天里出现视频卡片或文件卡片。

## 2. 群网盘转发

群网盘文件右键：

```text
转发
```

同样打开转发选择器。

## 3. 关键原则

```text
从网盘转发 = 秒发
不重新上传
```

系统只创建新的聊天消息，引用同一个 `file_object`。

# 十五、文件底层存储模型

## 1. 核心原则

同一份文件云端只保存一份。

```text
聊天消息引用 file_object
我的网盘目录引用 file_object
群网盘目录引用 file_object
```

如果用户多次发送同一文件，系统通过 hash 识别后秒传，不重复上传。

## 2. 推荐对象关系

```text
file_blob       真实物理文件
file_object     文件业务对象
drive_node      网盘目录节点
message_file    聊天消息文件引用
transfer_task   上传 / 下载任务
```

## 3. file_blob

用于去重和真实存储。

```ts
type FileBlob = {
  id: string;
  sha256: string;
  size: number;
  mimeType: string;
  ext: string;

  storageProvider: "oss" | "local" | "s3";
  refCount: number;

```

## 4. file_object

# 十六、上传技术策略

## 1. 小文件上传

小文件可直接上传。

```text
选择文件
计算基本信息
创建文件消息
上传文件
上传成功后更新消息状态
```

## 2. 大文件分片上传

大文件必须支持：

```text
分片上传
断点续传
失败重试
暂停继续
秒传
```

建议流程：

```text
选择文件
计算 sha256
请求后端检查是否已存在
如果存在：秒传完成
如果不存在：创建 multipart upload
分片上传
上传完成后合并
生成 file_object
消息状态更新为可下载 / 可播放
```

## 3. 分片大小

推荐：

```text
普通大文件：8MB / chunk
超大文件：16MB / chunk
```

## 4. 失败处理

上传失败时，文件卡片和传输窗口都显示失败状态。

```text
上传失败，点击重试
```

点击重试从已完成分片继续，不重新上传。

# 二十一、最终一句话方案

最终产品方案定为：

```text
聊天支持文件按钮、图片/视频按钮和拖拽上传；
视频在聊天内以独立视频卡展示，并使用内置 React 播放器；
大文件通过分片上传、断点续传和当前会话「传输」入口管理；
左侧主导航设为「网盘」，只进入我的网盘；
群网盘不放在左侧网盘里，而放在群聊标题栏右侧；
聊天文件不会自动进入网盘，必须由用户右键主动存入；
所有聊天文件、我的网盘文件、群网盘文件底层统一引用同一个 file_object；
同一份文件云端只保存一份，从网盘转发给好友或群聊时秒发，不重新上传。
```
