## 🔍 核心痛点评估与产品闭环思考

### Q1：上传方式怎么选？

* **评估：** **小孩子才做选择，成熟的IM系统必须两者兼备。**
* **UX设计：**
1. **主动入口：** 保留聊天工具栏已有的“文件夹”图标，点击唤起系统文件选择器。


2. **被动触发（拖拽上传）：** 用户将桌面的文件直接拖入聊天视口时，整个聊天消息区域必须瞬间平滑淡入一个**全覆盖式拖拽层（Dropzone Overlay）**，带有虚线边框和“*释放文件以发送*”的字样。这能极大提升大文件上传的直觉体验。



### Q2：视频播放怎么做？

* **评估：** **绝对不能调用本电脑自带的播放器！**
* **UX原因：** 外部原生播放器（如 Windows Media Player / QuickTime）会强行弹出一个巨大的独立新窗口，破坏聊天系统的沉浸感与视觉聚焦，且无法做到数据审计和播放状态回调。
* **设计建议：** 使用 React 自研一个支持标准 HTML5 `<video>` 的轻量级**灯箱播放器（Lightbox Player）**。对于网盘内的长视频，**甚至需要提供 1.5x / 2.0x 倍速播放功能**，方便老师快速预览教学录像。

### Q3 & Q4：大文件/音视频如何与“分用户/群网盘”联动？

* **闭环方案：** 聊天消息中传输的所有图片、视频、文件，在技术底层统一走一套 `file_object` 存储服务（元数据存 RDS，二进制文件存 OSS 分块存储）。网盘和聊天窗口实质上是**同一套文件底层在不同视图下的展现**。

## 🎨 核心功能场景模块化 UI/UX 设计

### 1. 聊天框多文件“拖拽接收层”交互 (Drag & Drop Overlay)

当鼠标拖拽文件悬停（`dragover`）在聊天面板时：

* **视觉规范：** 整个聊天消息流覆盖一层 `rgba(0, 123, 255, 0.08)` 的浅蓝半透明遮罩，边缘出现 `2px` 蓝色虚线框（`border: 2px dashed #007BFF`）。
* **中央提示：** 浮现一个带微动效的上传图标，下方标注：“*释放文件，直接发送至当前聊天*”。
* **多文件处理：** 允许一次性拖入多个文件，松开鼠标后（`drop`），自动进入发送队列。

---

### 2. 大文件/视频上传状态组件 (Chunky Upload Manager)

大文件（如 $\ge 100\text{MB}$ 的网课视频）必须支持**分片上传（Chunked Upload）与断点续传**。在聊天气泡中的 UI 表现如下：

```
+-------------------------------------------------------+
|  [📄 Excel 图标] 2026版中考生物复习题库.zip             |
|  245.8 MB / 450.0 MB   (正在上传... 3.2 MB/s)         |
|  +-------------------------------------------------+  |
|  |███████████████████████░░░░░░░░░░░░░░░░░░░░░░░░░|  |  <- 进度条 (54%)
|  +-------------------------------------------------+  |
|  [暂停]  [取消]                                       |
+-------------------------------------------------------+

```

* **状态机设计：** 提供 `上传中 | 暂停 | 失败重试 | 已完成` 四个原子状态。
* **微交互：** 点击“暂停”，向后端挂起上传流；由于网络闪断导致失败时，进度条整体变红，文字提示“*上传中断，点击重试*”，点击后即可从上一个已落盘的 Chunk 继续续传，避免老师重新上传的焦虑。

---

### 3. 内置式 React 视频灯箱播放器 (Inline Lightbox Video Player)

当老师在聊天流或网盘中点击一个 `.mp4` 视频时，直接在屏幕中央弹出一个置顶的现代微圆角悬浮层：

* **遮罩层：** 全屏暗化遮罩（`rgba(0, 0, 0, 0.65)`），让视觉完全聚焦在画面上。
* **播放器本体：** 宽度 `720px`，比例 `16:9`，圆角 `8px`，背景纯黑。
* **悬浮控制栏（毛玻璃质感）：** 鼠标滑入视频区域时自动从底部平滑浮现：
* **左侧：** `[播放/暂停按钮]` | `[当前时间 / 总时长 (03:14 / 12:45)]`
* **中间：** 流线型进度条，支持悬浮预览时间点（Hover Tooltip）。
* **右侧：** `[音量调节 Slider]` | `[倍速按钮 (点击展开 1.0x, 1.25x, 1.5x, 2.0x)]` | `[全屏切换]`。

### 交互核心特征与业务融合：

* **群网盘的权限制约：**
* 群网盘是天然的**组织教学资源库**。
* **UX细节：** 只有群主（老师）或管理员拥有“新建文件夹/删除/上传”的写权限，普通群成员（学生/家长）在群网盘视图下只有“下载”和“转发”的读权限，防止班级文件被误删。


* **网盘文件向聊天流的无缝流转（核心连通性）：**
* 在网盘列表的每一行右侧，常驻一个 **“发给好友/群”** 的快捷图标。
* 点击后，原地弹出我们在上一阶段重新设计的那个“精细化多选转发弹框”，勾选美女老师群或学生，点击确定，文件就会直接以统一的 `message_type: "file"` 或 `"video"` 气泡形式扔进聊天框。不需要重复上传，秒级发送！
